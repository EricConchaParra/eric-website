La capa operativa que Jira no trae.

PMToolkit agrega notas privadas, recordatorios, tags, señales de PRs y analytics accionables directamente sobre tu flujo diario. Menos seguimiento manual. Más contexto real para liderar.

Diseñado para PMs y Team Leads que necesitan visibilidad inmediata del sprint sin montar otro stack de herramientas alrededor.

Dentro de Jira

Un layer práctico para el trabajo diario.

PMToolkit no intenta reemplazar Jira. Lo completa con fricción mínima y con señales que hoy suelen quedar dispersas entre tabs, notas sueltas o seguimiento mental.

Drawer privado por issue

Anota riesgos, decisiones, contexto político o follow-ups sin exponerlos en comentarios públicos. El mismo contenido reaparece en dashboards y popup.

Tags y recordatorios reutilizables

Marca tickets con etiquetas personales, programa alarmas rápidas y vuelve a encontrarlos cuando abras Jira otra vez.

Indicadores donde haces scanning

La extensión añade badges de time in status, iconos de tracking y acciones de copiado en listas, tarjetas y vistas nativas.

Qué añade
  • Notas privadas y reminders
  • Tags compartidos en toda la extensión
  • Copy-link más limpio para Slack o Notion
  • Señales de PRs cuando GitHub está conectado
Popup y seguimiento

Tu cola personal de tickets no desaparece en Jira.

El popup funciona como bandeja de seguimiento rápida para revisar tickets tracked, buscar por contenido o tags, editar notas, copiar referencias y reactivar pendientes sin volver a abrir cada issue manualmente.

Búsqueda inmediata por ticket, contenido o tags.
Recordatorios persistentes vía `chrome.alarms` y notificaciones del navegador.
Demo mode separado para mostrar el producto sin tocar datos reales.
Analytics Hub

Un workspace analítico pensado para preguntas operativas.

El popup abre un hub con vistas dedicadas a sprint, performance, cierre y exportación histórica. No son dashboards decorativos: están orientados a decisiones de capacidad, riesgo y seguimiento.

Sprint Dashboard

Capacidad, ETA y salud del sprint por developer

Distribuye issues por persona, compara carga con horas disponibles, muestra velocity reciente y alerta cuando alguien queda overloaded o subutilizado.

GitHub PR Signals

El estado del ticket no es el estado del código

Cuando conectas GitHub, el dashboard enriquece tickets con PRs para ver qué realmente se está moviendo y saltar a revisión con un clic.

Señales rápidas

Preguntas que debería responderte en segundos.

¿Quién viene overloaded?

La vista por developer compara story points, horas remanentes y ETA para detectar sobrecarga antes de que el sprint te explote al final.

¿Qué ticket necesita seguimiento hoy?

Los reminders y tags reaparecen tanto en el drawer como en el popup, así que el pendiente no se pierde cuando cambias de contexto.

¿Qué se arrastra al siguiente sprint?

El Sprint Closure Report captura carryovers, scope creep y observaciones de cierre en formato más útil para screenshot o recap.

¿Quién está entregando qué ritmo?

Team Performance resume throughput, velocity y contribución por developer para dar contexto a la conversación, no solo a la sensación.

Instalación

Instalación simple, sin depender del admin de Jira.

PMToolkit corre como extensión de Chrome. Se descarga, se carga en modo desarrollador y queda lista para usar en tu navegador Chromium.

Sin tocar la configuración global de Jira Datos personales en storage del navegador Permisos acotados a Jira, Zoom y GitHub API opcional
Ver pasos de instalación Ver repositorio en GitHub

Descarga el `.zip`, descomprime y carga la carpeta en `chrome://extensions`.
